4. Minimale dataset & matching

Minimaal nodig om het model te laten draaien

  • Identiteit: reg (VRM — dé matchsleutel), make, model/derivative, year/plate, date_registered
  • Waardering inkoop: cap_clean_gbp (bied-anker), cap_retail_gbp, verwachte hamerprijs
  • Staat: grade, schaderegels (part, type, severity), declarations, MOT, service-historie, aantal sleutels, logbook
  • Gebruik: mileage + mileage_status, owners, fuel, transmission
  • Verkoopzijde (AT): retail-valuation, trade-valuation, Days to Sell, Retail Rating
  • Logistiek: veilinglocatie (voor transportafstand), sale_datetime
  • Kosten: volledige cost_config + repair_cost_model gevuld

Zonder deze velden geen score; alles daarbuiten (equipment, foto's, CO₂) is verrijking voor de detailweergave.

Waarop matchen we BCA ↔ Autotrader?

  • Primair (API, fase 1): exact op VRM + mileage → Auto Trader Connect geeft direct de valuation voor déze auto. Geen fuzzy matching nodig.
  • Fallback (scraping/PoC): make + model + derivative-trefwoorden + bouwjaar ±1 + km-band ±15% + fuel + transmission → mediaan van ≥5 listings; bij <5 listings band verbreden en match_confidence verlagen.
  • Conditie-correctie: AT-valuation is "retail ready" → schadekosten worden dus niet dubbel geteld: herstel zit aan de kostenkant, niet als korting op de verkoopwaarde.
  • Vastleggen: match-methode + confidence altijd opslaan in deal_scores.risk_flags / autotrader_matches.

Zelflerende loop. Kostenraming per auto = Σ over schaderegels: lookup (part_group, damage_type, severity_band) → user_cost ?? learned_cost ?? default_cost. Na herstel schrijft de gebruiker de factuurbedragen in repair_actuals; vanaf n ≥ 5 observaties per cel wordt learned_cost (voortschrijdend gemiddelde) de nieuwe default. Zo convergeert het model naar jouw werkelijke herstelpartner-prijzen.

6. Scoringsmodel

Netto marge = AT-retail-valuation − (verwachte hamerprijs + BCA-fees + herstelkosten + transport + opslag + recon)

Deal-score (0–100) = gewogen som van: marge% t.o.v. inkoop (40%), Days to Sell — sneller is beter (25%), Retail Rating (20%), risico-aftrek (15%: lage grade, declaraties, ontbrekende service-historie, mileage_status ≠ warranted, lage match_confidence).

Gewichten zijn config (in cost_config of aparte score_weights), zodat we ze kunnen kalibreren zodra er realisaties zijn. Verwachte hamerprijs fase 1 = cap_clean (conservatief); later leren uit werkelijke sold-prijzen.

7. Fase 1 — wat wel, wat niet

✅ In scope fase 1

  • Dagelijkse incrementele BCA-sync op de vaste filter (bestaande skills)
  • Volledige verrijking van nieuwe auto's incl. condition report
  • Autotrader Connect API: valuation + Days to Sell + Retail Rating per VRM (scraping alleen als tijdelijke fallback)
  • Kostenmodel met onderzochte defaults + user-override (UI in dashboard)
  • Zelflerende schadekosten: structuur + invoer realisaties (het "leren" is simpel gemiddelde, geen ML)
  • Scoring-engine + ranglijst-dashboard (HTML) op Supabase
  • RLS aanzetten met basispolicy zodra dashboard live gaat

⛔ Bewust buiten scope fase 1

  • Automatisch bieden/kopen — blijft altijd handmatig
  • NL-import (BPM, RDW, transport UK→NL) — model is UK-in/UK-uit; import is fase 2-module
  • Meerdere veilingbronnen (Manheim, Copart) — architectuur houdt er rekening mee (bron-kolom), bouwen niet
  • ML-prijsvoorspelling hamerprijs — eerst data verzamelen uit sold-lots
  • Multi-user, rollen, notificaties/alerts
  • Foto-analyse van schades (AI-vision) — condition report PDF is fase 1 de waarheid
